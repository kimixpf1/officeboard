## 长期优化储备（Opus 4.7 提出的优化建议，等以后有时间再做）

> 说明：以下为 Opus 4.7 在完整通读代码后给出的高性价比、低风险优化方向，原则是"绝对不能影响线上现有功能正常使用"。本轮暂不实施，留作后续排期。

### 功能体验（对标小智日历）
- **[中优先]** 一句话自然语言录入：在顶部 Toolbar 新增轻量单行输入框，复用现有 `js/kimi.js` / `js/ocr.js` 的大模型请求能力，针对单条日程定制 Prompt，回车后走现有预览确认流写入本地面板；对现有链路零干扰，显著降低碎片化事务录入摩擦
- **[低优先]** 日视图时间轴感增强：在现有日视图会议卡片左侧增加细时间进度线，在现有定时器中比对当前时间与 `item.time`，给进行中的会议卡片动态加 `.is-now` 类并加呼吸灯高亮，对标小智日历当前时间红线

### 代码架构（低风险防御性优化）
- **[中优先]** 逐步引入原生 ES Modules：将 `index.html` 中的顺序 `<script>` 标签改造为 `type="module"`，`utils.js` / `sync.js` / `app-date-view.js` 等按文件逐个补 `export` 与 `import`，彻底消除"函数找不到 / 全局变量被覆盖"的时序依赖隐患，不依赖任何构建工具
- **[中优先]** `ocr.js` 职责拆分：按"纯净 AI 层"与"业务合并层"物理切分为 `ai-parser.js`（只负责 Prompt 拼接 + 请求 + 返回标准化 JSON）与 `data-merger.js`（只负责与 IndexedDB 历史数据新增 / 合并 / 跳过比对），不改变既有业务链路，便于后续精准定位 AI 解析或合并逻辑的 Bug
- **[低优先]** 月度内存级缓存：在 `window` 单例中维护 `monthlyDataCache` 的 `Map`，周 / 月视图切换"天"时不再 `await db.getAll()`，仅在增 / 删 / 改 / `syncManager` 同步事件触发时清空缓存重拉，减少 IndexedDB 扫表次数与切换卡顿

### 落地原则
- 上述任一项实施时都必须遵循"先兼容、后优化"，保留旧链路作为回退
- 实施前先在本地完整复跑 OCR 识别 / 跨日期办文 / 会议完成态 / 倒数日同步 / 微信上传五条稳定链路，避免回归
- 每项单独提交、单独版本号递增，便于线上出现问题时精准回滚
